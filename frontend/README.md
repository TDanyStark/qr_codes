```js
  },

```

## Two-step login (dev)

This project includes a simple two-step login flow for development:

- `/` — enter your email. The app will generate a mock code and store it in localStorage.
- `/login/code` — enter the code you received. For development the generated code is logged to the browser console.

Files:

- `src/pages/LoginEmail.tsx` — email entry page
- `src/pages/LoginCode.tsx` — code entry page
- `src/lib/api.ts` — mock client that stores the pending email+code in localStorage

How to test:

1. Run the frontend dev server (e.g. `pnpm run dev` or `npm run dev`).
2. Open `/` and submit an email. Check the browser console for the generated code.
3. Navigate to `/login/code` and enter the code to complete login (redirects to `/users`).


You can also install [eslint-plugin-react-x](https://github.com/Rel1cx/eslint-react/tree/main/packages/plugins/eslint-plugin-react-x) and [eslint-plugin-react-dom](https://github.com/Rel1cx/eslint-react/tree/main/packages/plugins/eslint-plugin-react-dom) for React-specific lint rules:

```js
// eslint.config.js
import reactX from 'eslint-plugin-react-x'
import reactDom from 'eslint-plugin-react-dom'

export default tseslint.config([
  globalIgnores(['dist']),
  {
    files: ['**/*.{ts,tsx}'],
    extends: [
      // Other configs...
      // Enable lint rules for React
      reactX.configs['recommended-typescript'],
      // Enable lint rules for React DOM
      reactDom.configs.recommended,
    ],
    languageOptions: {
      parserOptions: {
        project: ['./tsconfig.node.json', './tsconfig.app.json'],
        tsconfigRootDir: import.meta.dirname,
      },
      // other options...
    },
  },
])
```
